# Documentation UI Generators

Documentation for `microframe/docs/ui.py` - Generators for interactive API documentation UIs.

## Overview

The `microframe/docs/ui.py` module provides classes that generate the HTML necessary to display interactive API documentation using popular tools like Swagger UI and ReDoc. These UIs fetch the OpenAPI specification (typically from the `/openapi.json` endpoint generated by `OpenAPIGenerator`) and render it in a user-friendly, explorable format.

By providing these generators, MicroFrame simplifies the process of integrating rich API documentation directly into your application.

### Purpose

-   **Interactive API Exploration**: Offers a visual and interactive way for developers to understand and test your API.
-   **Ease of Integration**: Abstracts the complexities of setting up Swagger UI and ReDoc by providing simple callable classes.
-   **Standardized Documentation**: Presents your API specification in a widely recognized and used format.
-   **Improved Developer Experience**: Enhances the overall experience for anyone consuming or working with your API.

## `SwaggerUI` Class

The `SwaggerUI` class generates the HTML page that embeds the Swagger UI, allowing for interactive exploration of your API based on the `/openapi.json` schema.

### Initialization

```python
class SwaggerUI:
    def __init__(self, title: str):
        self.title = title
```

**Parameters:**
-   `title` (str): The title to display in the browser tab and at the top of the Swagger UI page.

### Usage

Instances of `SwaggerUI` are callable and return an `HTMLResponse` containing the necessary HTML, CSS, and JavaScript to load Swagger UI. This UI automatically fetches its data from `/openapi.json`.

```python
# Typically used internally by the MicroFrame Application class:
# If you configure docs_url in your AppConfig, an instance of SwaggerUI
# will be created and served at that URL.

from microframe.docs.ui import SwaggerUI
from starlette.responses import HTMLResponse

# Example of how it works internally
swagger_ui_generator = SwaggerUI(title="My API Docs")
html_response: HTMLResponse = swagger_ui_generator()

# When your application serves this HTML, a browser will render Swagger UI.
```

## `ReDocUI` Class

The `ReDocUI` class generates the HTML page that embeds ReDoc, another popular tool for rendering OpenAPI specifications into a clean, single-page documentation format.

### Initialization

```python
class ReDocUI:
    def __init__(self, title):
        self.title = title
```

**Parameters:**
-   `title` (str): The title to display in the browser tab and at the top of the ReDoc page.

### Usage

Similar to `SwaggerUI`, instances of `ReDocUI` are callable and return an `HTMLResponse` that loads and configures ReDoc to display your API documentation from `/openapi.json`.

```python
# Typically used internally by the MicroFrame Application class:
# If you configure redoc_url in your AppConfig, an instance of ReDocUI
# will be created and served at that URL.

from microframe.docs.ui import ReDocUI
from starlette.responses import HTMLResponse

# Example of how it works internally
redoc_ui_generator = ReDocUI(title="My API ReDoc")
html_response: HTMLResponse = redoc_ui_generator()

# When your application serves this HTML, a browser will render ReDoc.
```

## Integration with MicroFrame Application

When initializing your `Application` instance with `AppConfig`, you can enable and configure these documentation UIs by setting the `docs_url` and `redoc_url` parameters.

```python
from microframe import Application, AppConfig

config = AppConfig(
    title="My Awesome API",
    version="1.0.0",
    description="A demonstration API for MicroFrame.",
    docs_url="/swagger",   # Swagger UI will be available at /swagger
    redoc_url="/redoc-docs", # ReDoc will be available at /redoc-docs
    openapi_url="/api/openapi.json" # The OpenAPI schema itself
)

app = Application(configuration=config)

# Now, when you run your app, navigating to /swagger will show Swagger UI,
# and /redoc-docs will show ReDoc. Both will use /api/openapi.json as their source.
```
